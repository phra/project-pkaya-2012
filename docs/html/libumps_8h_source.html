92 /* This function load a processor state from memory: there is no valid
93  * return value. New process may use status, EntryHI, pc and CAUSE as actual
94  * arguments if call is carefully built, since $4, $5, and $6 (a0, a1, a2)
95  * registers are not loaded from memory, but are passed as they are to
96  * new process, while $7 (a3) is loaded with CAUSE value from processor
97  * state in memory.
98  * Keep in mind that $2 (v0) register is used by routine itself and it is
99  * not loaded from memory image nor have a meaningful starting value for the
100  * new process this routine starts.
101  *
102  * This is NOT an atomic operation: the processor state is loaded register
103  * by register from memory, and at the end a BIOS routine will be called to
104  * load the critical CAUSE, STATUS, EntryHI and PC registers in one atomic
105  * operation: so, this call is interruptible (in a clean way) or cause a
106  * trap (for example, a memory access error if pointer is not correctly
107  * set).
108  * If called from user state, it will trap ONLY at BIOS call, loading the
109  * general registers with new/random values (if no other errors intervene);
110  * this will corrupt the calling process state, but it does not harm system
111  * security a bit (I thought I said you to use it only in kernel mode...)
112  */
113
